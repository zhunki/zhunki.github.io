2. Fuzzing with Code Fragments   link
   Christian Holler, Kim Herzig, Andreas Zeller
   Usenix Security 2012
   论文提出的LangFuzz使用语法随机生成有效程序，其特点是部分代码片段来自于以前导致异常行为的程序。LangFuzz在安全测试中非常有效。三个月内，在Mozilla JavaScript引擎发现了105个严重漏洞；在PHP解释器上发现了18个崩溃错误。

3. Domato   link
   Ivan Fratric
   2017
   Domato 是 Google Project Zero 开发的一个 DOM 模糊测试工具。它的核心是一个生成引擎，该引擎可以根据提供的上下文无关语法生成样本。这个工具的目的是通过自动化测试来发现和修复网页浏览器中的安全漏洞。Domato 主要用于安全研究和漏洞挖掘，帮助开发者和安全研究人员识别并修复可能被恶意利用的漏洞。

4. Skyfire: Data-driven seed generation for fuzzing   link
   Junjie Wang , Bihuan Chen , Lei Wei , Yang Liu
   S&P 2017
   Skyfire，一个数据驱动的种子生成工具，旨在提高模糊测试的效果和效率。Skyfire 通过分析现有的软件漏洞和程序错误来生成高质量的测试用例。它使用大量的数据和先进的算法来模拟可能的软件输入，以便更有效地发现潜在的安全漏洞和程序缺陷。这种方法使得模糊测试更加系统化和智能化，从而提高软件测试的质量和安全性。

5. CodeAlchemist: Semantics-Aware Code Generation to Find Vulnerabilities in JavaScript Engines   link
   HyungSeok Han, DongHyeon Oh, Sang Kil Cha
   NDSS 2019
   CodeAlchemist它通过一种独创的“语义感知装配”算法，能够生成既语义上又语法上正确的JavaScript代码片段。这与传统的模糊测试工具不同，后者通常只生成语法正确但语义上可能无效的测试用例。CodeAlchemist的这一创新方法有效地揭示了最新JavaScript引擎的多个漏洞。

6. Superion: Grammar-Aware Greybox Fuzzing   link
   Junjie Wang , Bihuan Chen , Lei Wei , Yang Liu
   ICSE 2019
   Superion 针对的是处理结构化输入（如XML和JavaScript）的程序。该方法的创新之处在于引入了基于语法的修剪策略和两种基于语法的变异策略，即增强的基于字典的变异和基于树的变异。这些策略通过使用解析测试输入的抽象语法树来进行操作。Superion作为AFL的扩展实现，不仅提高了代码覆盖率和发现新漏洞的能力，而且还发现了多个新的漏洞和安全漏洞。

7. NAUTILUS: Fishing for Deep Bugs with Grammars   link
   Cornelius Aschermann, Tommaso Frassetto, Thorsten Holz, Patrick Jauernig, Ahmad-Reza Sadeghi, Daniel Teuchert
   NDSS 2019
   NAUTILUS 通过结合语法分析和代码覆盖反馈，有效提升了对复杂输入程序的测试效率和深度。

8. Montage: A Neural Network Language Model-Guided JavaScript Engine Fuzzer   link
   Suyoung Lee, HyungSeok Han, Sang Kil Cha, Sooel Son
   Usenix Security 2020
   Montage 是一款利用神经网络语言模型（NNLM）指导的 JavaScript 引擎模糊测试工具。其创新点在于使用NNLM来生成更有效的JavaScript测试用例，以便发现JavaScript引擎中的复杂漏洞。Montage通过分析历史的JavaScript引擎相关的CVE，训练模型以识别JavaScript代码中的模式和共性。然后，它将JavaScript代码转换为抽象语法树（AST），并将这些AST分解成更小的子树用于模型训练。

9. Fuzzing JavaScript Engines with Aspect-preserving Mutation   link
   Soyeon Park, Wen Xu, Insu Yun, Daehee Jang, Taesoo Kim
   S&P 2020
   这项工作核心在于采用了保持代码特性的变异策略，以提高测试用例触发潜在漏洞的效率和效果。不同于传统模糊测试方法随机生成测试代码，这种方法在变异过程中保持了代码的语义和结构特性，能够生成更加精确和高效的测试用例。

10. SoFi: Reflection-Augmented Fuzzing for JavaScript Engines   link
   Xiaoyu He, Xiaofei Xie, Yuekang Li, Jianwen Sun, Feng Li, Wei Zou, Yang Liu, Lei Yu, Jianhua Zhou, Wenchang Shi, Wei Huo
   CCS 2021
   SoFi是一种新型的反射增强型JavaScript引擎模糊测试技术。其主要创新在于采用了细粒度程序分析和反射分析来生成语法和语义上有效的测试用例。这种方法可以识别可用变量，推断这些变量的类型，并对无效测试用例中的语法和语义错误进行自动修复。此外，SoFi通过反射分析来识别对象的未见属性和方法，从而在变异过程中使用，增强探索能力。

11. Token-Level Fuzzing   link
   Christopher Salls, Chani Jindal, Jake Corina, Christopher Kruegel, Giovanni Vigna
   Usenix Security 2021
   本工作创新之处在于它在Token级别上应用变异，而非传统的字节级或语法级别。这使得模糊测试器能够生成既符合语法又稍微偏离语法的输入，从而能够发现传统模糊测试技术无法发现的错误。

12. Jit-Picking: Differential Fuzzing of JavaScript Engines   link
   Lukas Bernhard, Tobias Scharnowski, Moritz Schloegel, Tim Blazytko, Thorsten Holz
   CCS 2022
   Jit-Picking专注于逻辑漏洞，该工具使用了差异性模糊测试，通过比较JavaScript引擎中解释器和即时编译器的执行结果来发现潜在的漏洞。

13. KOP-Fuzzer: A Key-Operation-based Fuzzer for Type Confusion Bugs in JavaScript Engines   link
   Lili Sun, Chenggang Wu, Zhe Wang, Yan Kang, and Bowen Tang
   COMPSAC 2022
   KOP-Fuzzer是一种针对JavaScript引擎类型混淆漏洞的模糊测试器。该工具基于人类经验，通过从源代码中提取关键操作来构建测试用例。相较于传统模糊测试器，KOP-Fuzzer在减少运行时错误的同时，有效提高了边缘覆盖率和关键操作覆盖率。

14. Fuzzilli: Fuzzing for JavaScript JIT Compiler Vulnerabilities   link
   Samuel Groß, Simon Koch, Lukas Bernhard, Thorsten Holz, Martin Johns
   NDSS 2023
   Fuzzilli基于自定义的中间语言FuzzIL，而不是改变 AST 或程序的其他语法元素，可以更直接地对程序的控制和数据流进行改变。该 IL 能够被翻译为 JavaScript 来执行。

15. FuzzJIT: Oracle-Enhanced Fuzzing for JavaScript Engine JIT Compiler   link
   Junjie Wang, Zhiyi Zhang, Shuang Liu, Xiaoning Du, and Junjie Chen
   Usenix Security 2023
   FuzzJIT基于一个核心思想：JIT编译器应该加速代码执行而不改变执行结果。FuzzJIT通过设计一种输入包装模板，能够主动激活JIT编译器，并使测试样本自身在执行过程中自动报告不一致行为。此外，研究者还开发了一套专门的变异策略，突出可能触发JIT漏洞的元素。
